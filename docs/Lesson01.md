
## Lesson 01: Internet of Things


### 1. IoT (Internet of Things) とは？

オン/オフスイッチのある、あらゆるデバイスをインターネット（および他の接続されたデバイス）に接続するという概念

#### IoTを構成する3つの要素

- デバイス (Devices)
- データ (Data)
- 接続性 (Connectivity)


### 2. なぜIoTが重要なのか？

IoTは、5Gネットワークの普及に伴い、日常生活の一部として生活の質と生産性を劇的に向上させ、新たなビジネスチャンスを創出する極めて重要な技術であるため


### 3. HTTPとは？

HTTPは Hyper Text Transfer Protocol の略称

#### 概要 (Introduction)

インターネット（ウェブ）は、大規模な分散型クライアント/サーバー情報システム

#### ワールドワイドウェブ・コミュニケーション (WORLD WIDE WEB COMMUNICATION)

ワールドワイドウェブ（WWW）の本質は、ウェブクライアントとウェブサーバー間の通信

- クライアント: 多くの場合ブラウザ（Chrome, Edge, Safari）であるが、あらゆる種類のプログラムやデバイスが含む
- サーバー: ほとんどの場合、クラウド上のコンピュータを指す

#### ハイパーテキスト・トランスファー・プロトコル (HTTP)

- HTTPは、インターネット（ウェブ）で最も普及しているアプリケーションプロトコル
- 非対称なリクエスト-レスポンス型: クライアントがリクエストを送り、サーバーがレスポンスを返す「プル型（Pull Protocol）」
- ステートレス (Stateless): 現在のリクエストは以前のリクエストの内容を保持しない
- 柔軟性: データ型や表現形式の交渉が可能であり、転送されるデータに依存せずにシステムを構築できる
- RFC2616の定義: 分散型、共同型、ハイパーメディア情報システムのための汎用的なプロトコルであり、ハイパーテキスト以外の拡張作業（ネームサーバー等）にも使用可能

#### HTTP リクエスト / レスポンス (HTTP REQUEST / RESPONSE)

1. クライアント（ブラウザ）がウェブサーバーへHTTPリクエストを送信 
2. ウェブサーバーがリクエストを受信。
3. サーバーがアプリケーションを実行してリクエストを処理 
4. サーバーがブラウザへHTTPレスポンス（出力）を返却 
5. クライアントがレスポンスを受信


### 4. HTMLとは

HTMLは、ウェブページを作成するための標準的なマークアップ言語

- HTML: Hyper Text Markup Language の略称
- 構造の記述: ウェブページの構造を記述
- 要素とタグ: HTMLは一連の「要素 (Elements)」で構成され、これらは「タグ (Tags)」によって表される
- ブラウザの役割: ブラウザはHTMLタグ自体を表示するのではなく、タグを使用してページのコンテンツをレンダリング（表示）する
